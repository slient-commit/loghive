Overview

LogHive Developer Docs

LogHive is a self-hosted log monitoring platform. This reference covers the full REST API, SDK usage for Node.js, Python, and .NET, and deployment configuration.

All API endpoints are prefixed with /api. Ingestion endpoints use API key authentication; all other endpoints require a JWT Bearer token from POST /api/auth/login.

Base URL

http
https://your-loghive-domain.com/api

Authentication headers

TypeHeaderUsed for
JWTAuthorization: Bearer <token>All authenticated endpoints
API Keyx-api-key: <key>Log ingestion only
Getting Started

Quickstart

1. Register an account

bash
curl -X POST https://your-domain.com/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@acme.com",
    "password": "securepassword",
    "first_name": "Jane",
    "last_name": "Doe",
    "organization_name": "Acme Corp"
  }'

2. Login and get a JWT

bash
curl -X POST https://your-domain.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@acme.com","password":"securepassword"}'

3. Create an app and generate an API key

bash
# Create app
curl -X POST https://your-domain.com/api/apps \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"name":"My API","environment":"production"}'

# Generate API key (use the UUID from the response above)
curl -X POST https://your-domain.com/api/apps/$UUID/keys \
  -H "Authorization: Bearer $TOKEN"

4. Send your first log

bash
curl -X POST https://your-domain.com/api/ingest \
  -H "x-api-key: lh_xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "level": "INFO",
    "message": "App started successfully",
    "tags": ["startup"],
    "metadata": {"version": "1.0.0"}
  }'

Authentication

Authentication

All auth endpoints are public (no token required) except GET /api/auth/me and PUT /api/auth/settings.

POST /api/auth/register Public

Create a new organization and admin account. Sends a verification email to confirm the address before login is allowed.

Body fieldTypeRequiredDescription
emailstring✓Admin email address.
passwordstring✓Minimum 8 characters.
first_namestring✓Admin first name.
last_namestring✓Admin last name.
organization_namestring✓Organization display name. Slug is auto-generated.
POST /api/auth/login Public

Authenticate and receive a JWT token. Email must be verified first.

Body fieldTypeRequiredDescription
emailstring✓
passwordstring✓
json — response
{
  "token": "eyJhbGc...",
  "user": { "id": 1, "email": "admin@acme.com", "role": "org_admin" },
  "organization": { "id": 1, "name": "Acme Corp" }
}
GET /api/auth/me JWT

Returns the current authenticated user with organization details.

GET /api/auth/verify-email?token=<token> Public

Verifies an email address using the token sent during registration. Returns a JWT on success so the user is immediately logged in.

POST /api/auth/forgot-password Public

Sends a password reset link. Always returns success to prevent email enumeration. Rate-limited to EMAIL_DAILY_LIMIT_PREAUTH (default 5) emails per address per day.

Body fieldTypeRequired
emailstring✓
POST /api/auth/reset-password Public

Resets the user's password using the token from the reset email. Token expires after 1 hour.

Body fieldTypeRequired
tokenstring✓
passwordstring✓
Log Ingestion

Log Ingestion

Ingestion endpoints authenticate via the x-api-key header using a per-app API key generated from the dashboard or API.

Logs with level FATAL trigger an instant email alert on their first occurrence per day (deduped via Redis). Users must have fatal email alerts enabled in their profile settings.
POST /api/ingest API Key

Send a single log entry.

Body fieldTypeRequiredDescription
levelstring✓DEBUG INFO WARN ERROR FATAL
messagestring✓Log message text.
tagsstring[]Searchable tags, e.g. ["auth","payments"].
metadataobjectArbitrary key-value pairs for drill-down filtering.
timestampISO 8601Defaults to current server time if omitted.
json — example body
{
  "level": "ERROR",
  "message": "Payment gateway timeout",
  "tags": ["payments", "stripe"],
  "metadata": { "orderId": "ord_abc123", "retryCount": 3 },
  "timestamp": "2025-06-20T10:00:00.000Z"
}
POST /api/ingest/batch API Key

Send up to 500 logs in a single request. The body is an array of log objects using the same fields as the single endpoint.

json — example body
[
  { "level": "INFO", "message": "Request received" },
  { "level": "WARN", "message": "Slow query", "metadata": { "ms": 1200 } }
]

Notifications

Notification Rules

Notification rules schedule email digests of your log data. Each rule runs on a cron schedule (UTC times), queries logs for the configured time range, and sends them in batched emails.

GET /api/notifications JWT

List all notification rules in the organization.

POST /api/notifications JWT

Create a new notification rule.

FieldTypeDefaultDescription
namestring—Rule display name. Required.
enabledbooleantrueWhether the rule runs.
schedule_timesstring[]—UTC times in HH:MM format, e.g. ["09:00","18:00"]. Required.
time_range_typestringlast_24hlast_24h or interval.
time_range_hoursnumber24Hours to look back when type is interval.
app_uuidsstring[][]Apps to include. Empty array = all apps.
log_levelsstring[]["ERROR","FATAL"]Log levels to include.
group_bystringnonenone level tag metadata
group_meta_keystring—Metadata key when group_by=metadata.
lines_per_emailnumber10Max rows per email. Excess go into follow-up emails.
email_delay_secondsnumber6Seconds between batched emails.
recipient_typestringorg_usersorg_users or custom.
recipientsstring[][]Email addresses when recipient_type=custom.
email_config_typestringsystemsystem (uses org email settings) or custom (rule-specific SMTP).
smtp_host / smtp_port / smtp_user / smtp_pass / smtp_fromstring—SMTP config when email_config_type=custom.
PUT /api/notifications/:uuid JWT

Update any fields of an existing rule. Reschedules the cron job automatically.

PATCH /api/notifications/:uuid/toggle JWT

Toggle a rule's enabled state. Returns {"enabled": true|false}.

POST /api/notifications/:uuid/test JWT

Enqueue an immediate digest for this rule. Email arrives within seconds.

DELETE /api/notifications/:uuid JWT

Delete a rule and stop its cron job.

Email Settings

Email Settings

Organization-level email configuration. Supports Resend or a custom SMTP server. Admin only for writes.

If no org-level settings are saved, the API falls back to RESEND_API_KEY and RESEND_FROM_EMAIL env vars. Individual notification rules can further override with a custom SMTP using email_config_type: "custom".
GET /api/settings/email JWT

Get the organization's email settings. Sensitive fields are masked — the response includes resend_api_key_set and smtp_pass_set booleans instead of the raw values.

PUT /api/settings/email JWT — Admin

Update email settings. Passing an empty string for secret fields (resend_api_key, smtp_pass) leaves the stored value unchanged.

FieldTypeDescription
providerstringresend or smtp.
resend_api_keystringResend API key. Falls back to env var if not set.
resend_from_emailstringSender address for Resend.
smtp_host / smtp_port / smtp_user / smtp_pass / smtp_fromstringSMTP config.
POST /api/settings/email/test JWT — Admin

Sends a test email to the authenticated user's address using the current org email settings.

SDK Reference

Installation

bash
npm install loghive-sdk

Usage

javascript
const LogHive = require('loghive-sdk');

const logger = new LogHive({
  apiKey:   'lh_xxxxxxxxxxxx',
  endpoint: 'https://your-loghive-instance.com',
});

// Direct send (awaitable)
await logger.info('User signed in', {
  tags: ['auth'],
  metadata: { userId: 42, ip: '1.2.3.4' },
});

await logger.error('Payment failed', { metadata: { orderId: 'ord_123' } });
await logger.fatal('Database connection lost');

// Queued — batched, auto-flushed every 5 s or 50 logs
logger.queue('warn', 'Slow query detected', { metadata: { ms: 1200 } });

// Graceful shutdown (flushes queue)
await logger.shutdown();

Methods

MethodDescription
debug / info / warn / error / fatalSend a log at the given level immediately.
send(level, message, opts)Generic send with explicit level.
queue(level, message, opts)Add to in-memory queue (non-blocking).
flush()Force-flush the queue immediately.
shutdown()Flush queue and clear the auto-flush timer.

Installation

bash
pip install loghive-sdk

Usage

python
from loghive import LogHive

logger = LogHive(
    api_key="lh_xxxxxxxxxxxx",
    endpoint="https://your-loghive-instance.com"
)

logger.info("User signed in", tags=["auth"], metadata={"user_id": 42})
logger.error("Payment failed", metadata={"order_id": "ord_123"})
logger.fatal("Database connection lost")

# Context manager — auto-flushes on exit
with LogHive(api_key="...", endpoint="...") as logger:
    logger.warn("Low disk space", metadata={"free_gb": 2.1})

Installation

Add the project reference or compiled DLL to your project. Targets .NET Standard 2.0+.

Usage

csharp
using LogHive;

var logger = new LogHiveClient(
    "lh_xxxxxxxxxxxx",
    "https://your-loghive-instance.com"
);

await logger.InfoAsync("App started");
await logger.ErrorAsync("Unhandled exception",
    tags: new[] { "critical" },
    metadata: new Dictionary<string, object> { ["userId"] = 99 });

// Batch queue — auto-flushed every 5 s or 50 logs
logger.Queue("warn", "High memory usage");

await logger.ShutdownAsync();

Deployment

Deployment

Docker Compose

The repository ships with a docker-compose.yml that builds and runs the API and React frontend behind nginx.

bash
# Clone and configure
git clone https://github.com/slient-commit/loghive
cd loghive
cp .env.example .env  # fill in your values

# Start
docker compose up -d

The frontend is served on port 80 (nginx). The API runs on port 3001 internally, proxied at /api/*. The landing page is served at / and the React app at all other routes.

MySQL, MongoDB, and Redis are not included in the compose file by default. Add them as services or connect to managed instances via environment variables.

Dokploy

LogHive is designed to deploy on Dokploy using Docker Compose with a shared external network.

  1. Create a Dokploy project and add a Docker Compose service pointing to your repository.
  2. Add all environment variables in the Dokploy environment tab.
  3. Set up MySQL, MongoDB, and Redis as separate Dokploy database services on the same dokploy-network.
  4. Point your domain to the app service — nginx serves the React frontend and proxies /api/* to the API.
The docker-compose.yml uses dokploy-network as an external network so all services can reach each other by container name.
